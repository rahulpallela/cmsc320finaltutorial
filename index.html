Data Collection

Our data is from the Environmental Protection Agency (EPA) an organization based in the United States that aims to preserve the Earth and ensure a clean, healthy communities for people in the United States. The dataset is derived from the EPA's Smart Location Database (SLD), which contains environmental information relating to environmental efficiency, accessibility, demographic statistics, and employment statistics all across the US. You can download the dataset here.

The dataset considers a single observation to be a Census Block Group (CBG), these CBG's are contained within Core-Based Statistical Areas (CBSA), which are typically metropolitan areas designated by the Office of Management and Budget. When CBSA's that are adjacent have a moderate amount of employment and social interchange they are considered to be a part of a Combined Statistical Area (CSA). For our purposes we will be using the CBSA as our main location identifier as it is the most specific identifier within the dataset.

The Smart Location Database contains information about many environmental indicators such as population information, employment information, and housing/automobile data. This data is compiled into 5 group each representing a factor group that contributes to walkability they are as follows.

  1. Density: These variables show that density of housing, employment, and population within the CBG.
  2. Employment and Housing Diversity: These variables contain information about the variety of job sectors and land use within the CBG.
  3. Urban Design: These variables quantify the traversability of the street network in the CBG.
  4. Transit Accessibility: These variables relate to the avaliability of convenient public transportation.
  5. Destination Accessibility: These variables measure the efficiency of commutes in the CBG.

To learn more about how each variable is calculated and the specific ways that data was collected by the EPA please read the Smart Location Database Technical Documentation and User Guide

We will provide a brief overview of the variables here nonetheless. The variables that inform the National Walkability Index are selected from the Smart Location Database (SLD), which has many more variables than the four selected for calculating the Walkability Index.

The four variables are as follows and are calculated on the block group level where a block group is the smallest area for which the US Census collects data. They are formed by boundaries of streets, roads, railroads, streams, and other features. More about block groups can be found here.

  1. Intersection density (D3B)

  2. Proximity to transit stops (D4A)

  3. Employment mix (D2B_E8MIXA)- mix of employment types in a block group such as retail, office, or industrial

  4. Employment and household mix (D2A_EPHHM)- mix of employment types with the amount of occupied housing units in the area also taken into account

def Mean_Squared_Error(predictions, actual):
    sum_sq_err = 0

    for i in range(0, len(predictions)):
        sq_err = (predictions[i]-actual[i])**2
        sum_sq_err += sq_err

    MSE = sum_sq_err/len(predictions)
    return MSE

This is a helper function to calculate the Mean Squared Error for each of our models.

#Ordinary Least Squares Linear Regression
def lin_reg(x_train, y_train, x_test, y_test):
    linreg = LinearRegression().fit(x_train, y_train)

    print('Intercept: ' + str(linreg.intercept_))
    print('D2A Coefficient: ' + str(linreg.coef_[0]))
    print('D2B Coefficient: ' + str(linreg.coef_[1]))
    print('D3B Coefficient: ' + str(linreg.coef_[2]))
    print('D4A Coefficient: ' + str(linreg.coef_[3]))

    predictions = linreg.predict(x_test)
    actual = y_test.to_list()

    MSE = Mean_Squared_Error(predictions, actual)
    print("Mean Squared Error: " + str(MSE))

    return MSE

For ordinary least squares linear regression we used the scikit-learn library to create a model that was fit with our training data and returns the intercept and 4 weights for the features. The model chooses weights such that the sum of the square errors is minimized.

# Gradient Descent (Lecture Code)
def grad_descent(x,y,epochs,alpha, x_test, y_test):
    m, n = x.shape
    theta = np.zeros(n)
    loss = np.zeros(epochs)

    for i in range(epochs):
        loss[i] = 0.5*np.linalg.norm(x.dot(theta)-y)**2
        gradient = np.transpose(x).dot(x.dot(theta)-y)
        theta = theta - alpha*gradient

    print('Intercept: ' + str(theta[0]))
    print('D2A Coefficient: ' + str(theta[1]))
    print('D2B Coefficient: ' + str(theta[2]))
    print('D3B Coefficient: ' + str(theta[3]))
    print('D4A Coefficient: ' + str(theta[4]))

    predictions = []
    actual = y_test.to_list()

    for index, row in x_test.iterrows():
        pred = theta[0] + theta[1]*x_test.at[index, 'D2A_Ranked'] + theta[2]*x_test.at[index, 'D2B_Ranked'] + theta[3]*x_test.at[index, 'D3B_Ranked'] + theta[4]*x_test.at[index, 'D4A_Ranked']
        predictions.append(pred)

    MSE = Mean_Squared_Error(predictions, actual)

    print("Mean Squared Error: " + str(MSE))

    return MSE, loss

The function above uses the gradient descent algorithm to optimize the weights for each of the variables to minimize the loss functions, in this case we elected to use the Mean Squared Error as our loss function. The gradient descent algorithm adjusts the weights for each epoch and adjusts them based on the gradient and the learning rate. Modifying the number of epochs and the learning rate can influence the accuracy of the weights. Having a learning rate that is too small will result in the weights changing by too little for each epoch and having a learning rate that is too large can result in sporadic and unintended behavior. If you would like to learn more about the Gradient Descent Algorithm here is a helpful deep dive%20is%20an,e.g.%20in%20a%20linear%20regression) into how it works.

def k_nearest_neighbors(k, x_train, y_train, x_test, y_test):

    knn = KNeighborsRegressor(n_neighbors = k)
    knn.fit(x_train, y_train)

    predictions = knn.predict(x_test)
    actual = y_test.to_list()

    print("Number of Neighbors: " + str(k))
    MSE = Mean_Squared_Error(predictions, actual)
    print("Mean Squared Error: " + str(MSE))
    score = knn.score(x_test,y_test)
    print("Accuracy: " + str(score))

    return MSE

The function implements a K Nearest Neighbors algorithm for regression this is done by finding the k observations with D2A, D2B, D3B, and D4A Ranks that are the closest to that of the test data then taking the average of the National Walkability Index Scores of these k observations and using that as the predicted value.

It is crucial that the value of K is chosen carefully since a K value that is too small may end up overfitting the data and a K value that is too large may end up underfitting the data. Typically we would use Cross Validation, such as GridSearch Cross Validation, to decide on the best value of K within a range of possible values. Due to the large size of our dataset and our computational limits we elected to use a simpler approach by taking the square root of the number of observations in our training data to get our value for K.
